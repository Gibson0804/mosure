# 项目配置

## 什么是项目配置？

项目配置是项目级别的设置，控制项目的各种行为和功能。每个项目都有独立的配置，包括 MCP 服务、API 设置、功能开关等。

### 核心功能

- **配置管理**：查看和修改项目配置
- **MCP 配置**：配置 MCP 服务的 Token 和开关
- **API 配置**：配置 API 访问控制和审计日志
- **维护操作**：修复表结构、清空项目数据

---

## 配置项

### 基础信息

- **项目展示名**：显示在导航或标题中的项目名称
- **项目描述**：简要说明项目用途或备注信息

### MCP 配置

- **启用 MCP 服务**：是否启用 MCP 服务
- **MCP Token**：MCP 服务的访问密钥（格式：项目前缀 + 16位随机 hex）

### API 配置

- **启用 IP 白名单**：是否启用 IP 白名单（仅影响 /open 开头的接口）
- **允许的 IP**：指定允许访问 API 的 IP 地址，例如 192.168.1.100 或 192.168.1.0/24
- **启用 CORS**：是否启用 CORS 跨域请求（仅影响 /open 开头的接口）
- **允许的 Origin**：指定允许访问 API 的域名，例如 http://example.com 或 https://*.example.com
- **启用审计日志**：是否记录审计日志
- **日志敏感字段屏蔽**：需要脱敏的字段列表（默认 password、token、secret）

## MCP 配置

### 生成 MCP Token

1. 在项目配置页面，找到"MCP 配置"部分
2. 点击"生成 Token"按钮
3. 系统会生成一个新的 Token
4. 复制 Token 和url到 MCP 客户端服务的配置中

### 启用/禁用 MCP 服务

1. 在项目配置页面，找到"MCP 配置"部分
2. 勾选或取消"启用 MCP 服务"开关
3. 点击"保存"

### 获取客户端配置

1. 在项目配置页面，找到"MCP 配置"部分
2. 点击"获取客户端配置"按钮
3. 系统会生成 MCP 客户端配置示例

---

## 维护操作

### 修复表结构

检查并创建缺失的表，补齐缺失字段。

1. 在项目配置页面，找到"维护"分类
2. 点击"修复表结构"按钮
3. 确认操作
4. 系统会自动检查并修复当前项目的数据库表结构

### 清空数据

清空当前项目的所有数据（表结构保留）。

> ⚠️ **警告**：此操作不可撤销，请谨慎操作。

1. 在项目配置页面，找到"维护"分类
2. 点击"清空数据"按钮
3. 确认操作
4. 系统会清空当前项目的所有数据

---

## 常见问题

### MCP Token 泄露了怎么办？

立即重新生成 Token，旧 Token 会自动失效。

### 配置修改会影响其他项目吗？

不会。每个项目有独立的配置，互不干扰。
