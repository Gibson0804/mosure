# 插件管理

## 什么是插件？

插件是扩展 Mosure 功能的模块，可以添加新的内容模型、云函数、菜单项、配置选项等。通过插件，你可以快速集成第三方服务或添加自定义功能，无需修改核心代码。

### 核心功能

- **插件市场**：浏览和安装可用插件
- **插件管理**：启用、禁用、卸载插件
- **自动集成**：插件安装后自动集成到系统
- **版本管理**：查看和更新插件版本

---

## 插件市场

### 浏览插件

1. 进入"插件管理"页面
2. 点击"插件市场"按钮
3. 浏览可用的插件

### 插件信息

每个插件显示以下信息：

- **插件名称**：插件的显示名称
- **插件描述**：插件的功能说明
- **插件版本**：当前版本号
- **开发者**：插件开发者信息
- **安装数量**：插件被安装的次数
- **评分**：插件的用户评分

---

## 安装插件

### 基本步骤

1. 在插件市场找到要安装的插件
2. 点击"安装"按钮
3. 查看插件的详细信息
4. 确认安装

### 安装过程

插件安装过程中，系统会：

1. 下载插件文件
2. 检查依赖项
3. 执行安装脚本
4. 注册插件资源（模型、函数、菜单等）
5. 完成安装

### 安装后

插件安装完成后，可能会：

- 添加新的内容模型
- 添加新的云函数
- 添加新的菜单项
- 添加新的配置选项
- 修改系统行为

## 插件包格式

### ZIP 格式

插件使用 ZIP 格式打包，包含以下内容：

```
plugin-name
├── plugin.json          # 插件配置文件
├── PluginName.php       # 插件主类文件
├── functions/           # 云函数（可选）
│   ├── endpoints/       # Web 函数目录
│   │   ├── function1.json
│   │   └── function2.json
│   ├── hooks/           # 触发函数目录
│   │   ├── hook1.json
│   │   └── hook2.json
│   ├── variables.json   # 变量定义（可选）
│   ├── triggers.json    # 触发器配置（可选）
│   └── schedules.json   # 定时任务（可选）
├── menus/               # 菜单配置（可选）
│   ├── menu1.json
│   └── menu2.json
├── models/              # 内容模型（可选）
│   ├── model1.json
│   └── model2.json
├── data/                # 内容模型数据（可选）
│   └── model1/
│       ├── content1.json
│       └── content2.json
├── src/                 # PHP 源代码文件（可选）
│   └── CustomHandler.php
└── frontend/            # 前端资源（可选）
    ├── index.html
    └── assets/
```

### plugin.json 结构
```json
{
  "id": "plugin-name@1.0.0",
  "version": "1.0.0",
  "name": "插件名称",
  "description": "插件功能说明",
  "author": "开发者名称",
  "homepage": "插件主页",
  "type": "template",
  "tags": ["标签1", "标签2"],
  "compatible": {
    "min_version": "1.0.0",
    "max_version": "2.0.0"
  },
  "dependencies": [],
  "has_frontend": false,
  "has_src": false,
  "snapshot": false,
  "provides": {
    "models": [],
    "functions": {
      "endpoints": [],
      "hooks": [],
      "variables": false,
      "triggers": false,
      "schedules": false
    },
    "menus": [],
    "data": []
  }
}
```

### 插件主类

插件必须实现 `PluginInterface` 接口，包含以下方法：

```php
<?php

namespace Plugins\plugin_name;

use Plugins\PluginInterface;

class PluginName implements PluginInterface
{
    public function getId(): string
    {
        return 'plugin-name';
    }

    public function getName(): string
    {
        return '插件名称';
    }

    public function getVersion(): string
    {
        return '1.0.0';
    }

    public function getConfig(): array
    {
        return [
            'description' => '插件功能说明',
            'author' => '开发者名称',
            'type' => 'template',
            // ... 其他配置
        ];
    }

    public function install(string $projectId): void
    {
        // 安装逻辑
    }

    public function uninstall(string $projectId): void
    {
        // 卸载逻辑
    }

    public function onBeforeInstall(string $projectId): void
    {
        // 安装前钩子
    }

    public function onAfterInstall(string $projectId): void
    {
        // 安装后钩子
    }
}
```

### 函数文件格式

```json

  "name": "函数名称",

  "description": "函数描述",
  "type": "endpoint",

  "enabled": true
```

```json
  "description": "触发函数描述",
  "type": "hook",
  "code": "function code here",

```

```json
      "field": "title",
      "label": "标题",
      "type": "input"
    }
  ],
  "settings": {},
  "subject_content": {},
  "list_show_fields": ["title", "created_at"]
}
```

### src 目录

`src/` 目录用于存放自定义 PHP 类文件，例如：

- 自定义处理器类
- 辅助工具类
- 服务类

这些文件会在插件安装时自动加载。

### frontend 目录

`frontend/` 目录用于存放编译后的前端资源，例如：

- 编译后的 JavaScript 文件
- CSS 样式文件
- 静态资源（图片、字体等）

如果 `has_frontend` 为 `true`，前端资源会自动集成到系统中。

> ⚠️ **注意**：frontend 目录只支持编译后的文件，不支持直接存放 Vue、React 等源代码文件。

## 上传插件

### 上传步骤

1. 进入"插件管理"页面
2. 点击"上传插件"按钮
3. 选择插件 ZIP 文件
4. 系统会自动：
   - 解压插件文件
   - 验证 plugin.json 格式
   - 检查插件依赖
   - 安装插件到系统
5. 上传成功后，插件会显示在插件列表中

### 上传要求

- 文件格式：ZIP
- 文件大小：最大 100MB
- 必须包含 `plugin.json` 文件
- 必须包含插件主类文件

## 插件管理

### 启用/禁用插件

1. 在插件列表中，点击插件的"启用/禁用"按钮
2. 确认操作

禁用插件后，该插件提供的功能将暂时不可用。

### 卸载插件

1. 在插件列表中，点击插件的"卸载"按钮
2. 确认卸载

> ⚠️ **警告**：卸载插件会删除该插件的所有数据和配置，此操作不可恢复。

### 更新插件

1. 在插件列表中，点击插件的"检查更新"按钮
2. 如果有新版本，点击"更新"按钮
3. 确认更新

### 查看插件详情

1. 在插件列表中，点击插件名称
2. 查看插件的详细信息，包括：
   - 功能说明
   - 配置选项
   - 使用文档
   - 更新日志

## 常见问题

### 插件会影响系统性能吗？

插件的质量会影响系统性能。建议只安装可信的插件，并定期检查插件的性能影响。

### 插件之间会有冲突吗？

有可能。如果两个插件修改了同一个功能，可能会产生冲突。建议在安装前查看插件的兼容性说明。

### 如何开发自己的插件？

见[插件开发文档](/docs/5.开发者指南/04-插件开发)。

### 插件会自动更新吗？

不会。插件需要手动更新。建议定期检查插件更新，获取最新功能和修复。
