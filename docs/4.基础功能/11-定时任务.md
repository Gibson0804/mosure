# 定时任务

## 什么是定时任务？

定时任务（Cron Job）允许你按照预定的时间规则自动执行任务，如每天备份数据、每小时同步数据、每周生成报告等。定时任务基于 Cron 表达式定义执行时间。

### 核心功能

- **Cron 表达式**：使用标准 Cron 表达式定义执行时间
- **任务管理**：创建、编辑、启用/禁用、删除定时任务
- **执行记录**：查看任务的执行历史和日志
- **立即执行**：手动触发任务立即执行

---

## Cron 表达式

Cron 表达式由 5 个字段组成，分别表示分钟、小时、日期、月份、星期：

```
* * * * *
│ │ │ │ │
│ │ │ │ └─ 星期 (0-7, 0和7都表示周日)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日期 (1-31)
│ └─────── 小时 (0-23)
└───────── 分钟 (0-59)
```

### 常用示例

| 表达式 | 说明 |
|--------|------|
| `* * * * *` | 每分钟执行 |
| `0 * * * *` | 每小时执行 |
| `0 0 * * *` | 每天午夜执行 |
| `0 0 * * 0` | 每周日午夜执行 |
| `0 0 1 * *` | 每月1号午夜执行 |
| `0 9 * * 1-5` | 工作日早上9点执行 |
| `*/5 * * * *` | 每5分钟执行 |
| `0 */2 * * *` | 每2小时执行 |

## 创建定时任务

### 基本步骤

1. 进入"定时任务"页面
2. 点击"新建任务"按钮
3. 填写任务信息：
   - **任务名称**：任务的显示名称
   - **Cron 表达式**：定义执行时间
   - **执行内容**：选择要执行的操作
4. 点击"保存"完成创建

### 执行内容

定时任务可以执行以下操作：

- **调用云函数**：执行指定的云函数
- **发送通知**：发送邮件或短信通知
- **数据同步**：同步数据到外部系统
- **数据清理**：清理过期数据
- **生成报告**：生成统计报告

## 定时任务管理

### 编辑任务

1. 在任务列表中，点击任务的"编辑"按钮
2. 修改任务信息
3. 点击"保存"

### 启用/禁用任务

1. 在任务列表中，点击任务的"启用/禁用"按钮
2. 确认操作

禁用任务后，该任务将不再按计划执行。

### 删除任务

1. 在任务列表中，点击任务的"删除"按钮
2. 确认删除

### 立即执行

1. 在任务列表中，点击任务的"立即执行"按钮
2. 任务会立即执行一次，不影响正常的执行计划

### 查看执行记录

1. 在任务列表中，点击任务的"执行记录"按钮
2. 查看任务的执行历史，包括：
   - 计划执行时间
   - 实际执行时间
   - 执行状态
   - 执行结果
   - 错误信息（如果有）

## 常见问题

### 定时任务会自动执行吗？

是的。系统会每分钟检查一次是否有定时任务需要执行，如果有则自动执行。

### 定时任务执行失败会重试吗？

（待补充：重试机制）


### 定时任务会影响性能吗？

定时任务的执行是异步的，不会影响主流程的性能。但建议合理配置任务，避免过多任务同时执行。

### 如何测试定时任务？

1. 使用"立即执行"功能测试任务
2. 查看执行记录，了解任务的实际执行情况
3. 检查 Cron 表达式是否正确

### 定时任务可以调用云函数吗？

可以。定时任务可以调用任何云函数，实现复杂的业务逻辑。
