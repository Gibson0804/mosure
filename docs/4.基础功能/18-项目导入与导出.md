# 项目导入与导出

## 什么是项目导入与导出？

项目导入与导出功能允许你将整个项目或部分内容导出为 ZIP 文件，然后在其他地方导入。这对于项目备份、数据迁移、团队协作等场景非常有用。

### 核心功能

- **项目导出**：导出项目内容（模型、函数、菜单、数据等）
- **项目导入**：从导出文件导入项目
- **选择性导出**：选择要导出的内容类型
- **冲突处理**：检测数据冲突并提供多种处理方式

---

## 项目导出

### 访问入口

进入"管理" → "项目导出/导入"页面

### 导出步骤

1. 选择要导出的内容：
   - **内容模型**：选择要导出的模型
   - **云函数**：选择要导出的 Web 函数和触发函数
   - **菜单**：选择要导出的自定义菜单
   - **内容数据**：勾选是否导出内容数据
2. 点击"开始导出"按钮
3. 系统生成 ZIP 文件并自动下载

### 导出内容说明

| 内容类型 | 说明 | 文件位置 |
|---------|------|---------|
| 内容模型 | 模型定义和字段配置 | `models/` 目录 |
| Web 函数 | API 接口函数 | `functions/endpoints/` 目录 |
| 触发函数 | 触发器函数 | `functions/hooks/` 目录 |
| 变量 | 函数变量定义 | `functions/variables.json` |
| 定时任务 | 定时执行的任务 | `functions/schedules.json` |
| 菜单 | 自定义菜单配置 | `menus/` 目录 |
| 内容数据 | 模型中的内容数据 | `data/` 目录 |

## 项目导入

### 导入步骤

1. 进入"管理" → "项目导出/导入"页面
2. 点击"导入项目"按钮
3. 上传导出文件（ZIP 格式）
4. 系统解析文件并显示导入预览
5. 检查导入预览，确认要导入的内容
6. 点击"确认导入"
7. 等待导入完成
8. 导入成功后自动刷新页面

### 冲突检测

导入前，系统会检测以下冲突：

- **模型名称冲突**：检测导入的模型名称是否已存在
- **函数名称冲突**：检测导入的函数名称是否已存在
- **菜单 key 冲突**：检测导入的菜单 key 是否已存在

如果存在冲突，系统会在导入预览中提示，但不会自动处理冲突。

### 数据验证

导入前，系统会验证以下内容：

- **文件格式**：确认是有效的 ZIP 文件
- **manifest.json**：确认清单文件格式正确
- **数据完整性**：确认数据完整无损
- **依赖关系**：确认依赖关系正确
- **冲突检测**：检测数据冲突

## 导出文件格式

### ZIP 格式

项目导出使用 ZIP 格式，包含以下内容：

```
project-export.zip
├── manifest.json          # 清单文件
├── models/                # 内容模型定义
│   ├── model1.json
│   └── model2.json
├── functions/             # 云函数
│   ├── endpoints/         # Web 函数目录
│   │   ├── function1.json
│   │   └── function2.json
│   ├── hooks/             # 触发函数目录
│   │   ├── hook1.json
│   │   └── hook2.json
│   ├── variables.json     # 变量定义
│   └── schedules.json     # 定时任务
├── menus/                 # 菜单配置
│   ├── menu1.json
│   └── menu2.json
└── data/                  # 内容数据（可选）
    └── model1/
        ├── content1.json
        └── content2.json
```

### manifest.json 结构

```json
{
    "package_version": "1.0.0",
    "exported_at": "2026-01-19T15:47:02+08:00",
    "contents": {
        "models": [
            "todo",
            "statistics"
        ],
        "functions": {
            "endpoints": [
                "day_undo"
            ],
            "hooks": [
                "end_time",
                "send_msg"
            ],
            "variables": true,
            "triggers": true,
            "schedules": true
        },
        "menus": [
            "menu_a9a4ecab"
        ],
        "data": [
            "todo"
        ],
        "media": []
    }
}
```

### 模型文件格式

`models/model1.json` 示例：

```json
{
  "id": 1,
  "name": "文章",
  "description": "文章内容模型",
  "table_name": "article",
  "mold_type": "content",
  "fields": [
    {
      "field": "title",
      "label": "标题",
      "type": "input"
    }
  ],
  "settings": {},
  "subject_content": {},
  "list_show_fields": ["title", "created_at"]
}
```

`menus/menu1.json` 示例：

## 常见问题

### 导出文件有多大？

导出文件的大小取决于项目数据量。通常在几 MB 到几百 MB 之间。

### 导入会覆盖数据吗？

取决于冲突处理方式：
- 选择"覆盖"会覆盖现有数据
- 选择"跳过"不会影响现有数据
- 选择"重命名"会创建新的数据

### 可以部分导入吗？

可以。在导出时可以选择要导出的内容，导入时会根据 manifest.json 导出清单导入对应的内容。

### 导入失败怎么办？

查看系统返回的错误信息，常见原因包括：

- 文件格式错误或损坏
- manifest.json 格式错误
- 数据不完整
- 依赖关系冲突
- 数据库约束冲突

### 导出文件可以分享吗？

可以。导出文件可以分享给其他用户，他们可以使用导入功能恢复项目。但请注意保护敏感信息（如 API 密钥、数据库配置等）。
