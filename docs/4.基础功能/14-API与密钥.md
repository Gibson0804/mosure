# API 与密钥

## 什么是 API 密钥？

API 密钥（API Key）是访问 Mosure 开放 API 的凭证。每个密钥都有独立的权限和配置，可以针对不同的应用或服务创建不同的密钥。

### 核心功能

- **密钥管理**：创建、编辑、删除、启用/禁用密钥
- **权限控制**：为每个密钥配置不同的权限范围
- **访问监控**：查看密钥的调用日志和统计
- **安全控制**：定期更换密钥，限制访问频率

---

## 创建 API 密钥

### 基本步骤

1. 进入"API 与密钥"页面
2. 点击"新建密钥"按钮
3. 填写密钥信息：
   - **密钥名称**：密钥的显示名称，如"移动端应用"
   - **描述**：密钥的用途说明（可选）
   - **访问频率限制**：设置每分钟的最大调用次数（默认 1000 次/分钟，范围 1-10000）
   - **过期时间**：设置密钥的过期时间（可选）

4. 点击"保存"
5. 系统会生成一个密钥字符串，请妥善保存



---

## 使用 API 密钥

### 请求头

在调用开放 API 时，需要在请求头中携带 API 密钥：

```
X-API-Key: your_api_key_here
```

或者使用 Bearer Token 格式：

```
Authorization: Bearer your_api_key_here
```

### 示例请求

```bash
curl -X GET "http://127.0.0.1:9445/open/blog/content/list/article" \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json"
```

## 密钥管理

### 编辑密钥

1. 进入"API 与密钥"页面
2. 点击密钥列表中的"编辑"按钮
3. 修改密钥信息
4. 点击"保存"

> ⚠️ **注意**：密钥字符串创建后不可修改，如需更换请重新生成。

### 启用/禁用密钥

1. 进入"API 与密钥"页面
2. 点击密钥列表中的"启用/禁用"按钮
3. 确认操作

禁用密钥后，使用该密钥的 API 请求将失败。

### 删除密钥

1. 进入"API 与密钥"页面
2. 点击密钥列表中的"删除"按钮
3. 确认删除

> ⚠️ **警告**：删除密钥后，使用该密钥的应用将无法访问 API。

### 重新生成密钥

如果密钥泄露或需要更换：

1. 进入"API 与密钥"页面
2. 点击密钥列表中的"重新生成"按钮
3. 系统会生成一个新的密钥字符串
4. 复制新的密钥字符串

> ⚠️ **注意**：重新生成功能只生成新密钥字符串，不会自动保存到数据库。你需要手动创建新密钥并删除旧密钥。旧密钥在删除前仍然有效。

## 访问监控

### 调用日志

你可以查看每个密钥的调用日志：

1. 进入"API 与密钥"页面
2. 点击密钥列表中的"查看日志"按钮
3. 查看调用记录，包括：
   - 调用时间
   - 调用的 API
   - 请求参数
   - 响应状态
   - 响应时间

### 调用统计

查看密钥的调用统计：

- 总调用次数
- 成功调用次数
- 失败调用次数
- 平均响应时间

## 常见问题

### API 密钥可以共享吗？

不建议。每个应用或服务应该使用独立的 API 密钥，这样便于管理和撤销权限。

### 密钥泄露了怎么办？

立即重新生成密钥，旧密钥会自动失效。同时检查调用日志，确认是否有异常调用。

### 如何限制 API 访问频率？

在创建或编辑密钥时，可以设置访问频率限制，如每分钟最多调用 100 次。

### 密钥会过期吗？

可以设置过期时间。如果不设置过期时间，密钥将永久有效（除非手动禁用或删除）。系统会定期清理过期的密钥。
