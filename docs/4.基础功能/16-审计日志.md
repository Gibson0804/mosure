# 审计日志

## 什么是审计日志？

审计日志记录了通过开放 API（/open 接口）执行的所有操作，包括操作人、操作时间、操作类型、操作对象等。通过审计日志，你可以追踪 API 的使用情况，排查问题，确保数据安全。

### 核心功能

- **操作记录**：自动记录所有 open 接口操作
- **日志查询**：按条件筛选和搜索日志
- **日志详情**：查看操作的详细信息
- **数据安全**：敏感信息脱敏处理

---

## 日志类型

Mosure 记录以下类型的操作日志（仅限 open 接口）：

| 操作类型 | 说明 | 示例 |
|---------|------|------|
| 内容操作 | 内容的创建、更新、删除 | 创建文章、修改产品信息 |
| 单页操作 | 单页内容的更新 | 更新关于我们页面 |
| 媒体操作 | 媒体文件的上传、删除 | 上传图片、删除视频 |
| 函数调用 | 云函数的调用 | 调用自定义函数 |

---

## 访问审计日志

### 基本步骤

1. 进入管理后台
2. 点击左侧菜单的"审计日志"
3. 查看所有 open 接口操作日志

---

## 常见问题

### 审计日志会记录哪些操作？

只记录通过 open 接口执行的操作，包括内容操作、单页操作、媒体操作、函数调用等。不记录管理后台的操作。

### 审计日志会存储 API 密钥吗？

不会。审计日志只记录 API 密钥的 ID，不存储原始密钥字符串，避免敏感信息泄露。

### 审计日志可以删除吗？

可以。管理员可以删除审计日志，但建议保留重要操作的日志以便追溯。

### 如何关闭审计日志？

在系统配置中可以关闭审计日志。

### 审计日志会占用大量空间吗？

会。审计日志会占用一定的数据库空间，建议定期清理过期日志。

### 审计日志可以用于什么？

- 追踪 API 调用历史
- 排查 open 接口问题
- 数据审计
- 安全监控
