# 触发器

## 什么是触发器？

触发器是一种事件响应机制，当特定事件发生时自动执行绑定的触发函数。触发器通过监听系统事件（如内容创建、更新、删除，或函数执行）来自动触发自定义的业务逻辑。

### 核心功能

- **事件监听**：监听内容操作和函数执行事件
- **自动触发**：事件发生时自动执行绑定的触发函数
- **执行记录**：查看触发器的执行历史
- **灵活配置**：支持多种事件类型和触发条件

---

## 触发器类型

Mosure 支持以下触发器类型：

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 内容模型 | 监听指定模型的内容操作事件 | 自动处理、数据同步、数据清洗、自动同步三方平台 |
| 内容单页 | 监听指定单条内容的操作事件 | 单页内容联动 |
| 函数执行 | 监听云函数的执行事件 | 函数调用链、日志记录 |

---

## 创建触发器

### 基本步骤

1. 进入"触发器"页面
2. 点击"新建触发器"按钮
3. 填写触发器信息：
   - **触发器名称**：触发器的显示名称
   - **触发器类型**：选择触发器类型
   - **监听事件**：选择要监听的事件
   - **绑定对象**：选择要监听的模型、内容或函数
   - **执行函数**：选择要执行的触发函数
4. 点击"保存"完成创建

---

## 监听事件

触发器可以监听以下事件：

### 内容事件

- **content.before_create**：内容创建前
- **content.after_create**：内容创建后
- **content.before_update**：内容更新前
- **content.after_update**：内容更新后
- **content.before_delete**：内容删除前
- **content.after_delete**：内容删除后

### 函数事件

- **function.before_execute**：函数执行前
- **function.after_execute**：函数执行后

---

## 执行函数

触发器执行时，会调用绑定的触发函数（type=hook, runtime=php）。

### 函数参数

触发函数会接收以下参数：

- **`$payload`**：事件数据，包含事件相关信息
- **`$env`**：项目环境变量
- **`$ctx`**：执行上下文
- **`$event`**：触发的事件名称
- **`$Http`**：HTTP 请求工具
- **`$db`**：数据库操作工具

### 示例

```php
<?php 
// $payload
// {
// 			"mold_id": 3,
// 			"id": 2,
// 			"before": {
// 				"id": 2,
// 				"title": "吃饭",
// 				"updated_by": null,
// 				"created_by": null,
// 				"content_status": null,
// 				"created_at": "2026-01-18 20:00:26",
// 				"updated_at": "2026-01-18 21:16:21",
// 				"deleted_at": null,
// 				"completed": 0,
// 				"finish_time": null
// 			},
// 			"after": {
// 				"id": 2,
// 				"title": "吃饭",
// 				"updated_by": null,
// 				"created_by": null,
// 				"content_status": null,
// 				"created_at": "2026-01-18 20:00:26",
// 				"updated_at": "2026-01-18 21:16:21",
// 				"deleted_at": null,
// 				"completed": 1,
// 				"finish_time": null
// 			}
// 		}
// 触发函数：当 completed 从 0 变为 1 时，记录完成时间
$before = $payload['before'] ?? [];
$after = $payload['after'] ?? [];


// 检查 completed 是否从 0 变为 1
$oldCompleted = (int)($before['completed'] ?? 0);
$newCompleted = (int)($after['completed'] ?? 0);
Log::info('TriggerService res time ' . date('Y-m-d H:i:s')  . ' offset_time ' . $env['offset_time']. ' info ' . json_encode($payload['after']));


if ($oldCompleted === 0 && $newCompleted === 1) {
    // 第一次完成，且 finish_time 为空时记录完成时间
    if (empty($after['finish_time'])) {
        $id = (int)($after['id'] ?? 0);
        $offsetFinishTime = date('Y-m-d H:i:s', time() + $env['offset_time'] * 60);
        $res = $db->update($prefix . '_mc_todo', ['finish_time' => $offsetFinishTime], ['id' => $id]);
    }
}

Log::info('TriggerService res time ' . date('Y-m-d H:i:s')  . ' offset_time ' . $offsetFinishTime. ' info ' . json_encode($payload['after']));
return ['ok' => true];
```

## 触发器管理

### 编辑触发器

1. 在触发器列表中，点击触发器的"编辑"按钮
2. 修改触发器配置
3. 点击"保存"

### 启用/禁用触发器

1. 在触发器列表中，点击触发器的"启用/禁用"按钮
2. 确认操作

禁用触发器后，该触发器将不再执行。

### 删除触发器

1. 在触发器列表中，点击触发器的"删除"按钮
2. 确认删除

### 查看执行记录

1. 在触发器列表中，点击触发器的"执行记录"按钮
2. 查看触发器的执行历史，包括：
   - 触发时间
   - 触发事件
   - 执行状态
   - 执行结果
   - 错误信息（如果有）

## 常见问题

### 触发器会无限循环吗？

正常的使用和配置不会。触发器是事件驱动的，只在特定事件发生时执行，不会无限循环。

### 触发器执行失败会重试吗？

不会。触发器执行失败后不会自动重试，需要手动处理。

### 可以创建多少个触发器？

目前没有数量限制。

### 触发器会影响性能吗？

触发器的执行是同步的，但建议合理配置触发器，避免过多不必要的触发。

### 如何调试触发器？

1. 查看触发器的执行记录
2. 检查触发函数的代码逻辑
3. 在执行记录中查看错误信息
4. 使用日志记录调试信息
