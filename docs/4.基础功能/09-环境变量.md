# 环境变量

## 什么是环境变量？

环境变量是项目级别的配置参数，用于存储敏感信息或配置数据，如 API 密钥、数据库连接信息等。环境变量可以在云函数中使用，避免将敏感信息硬编码在代码中。

### 核心功能

- **变量管理**：创建、编辑、删除环境变量
- **安全存储**：敏感信息加密存储
- **函数访问**：云函数可以读取环境变量
- **作用域控制**：每个项目有独立的环境变量

---

## 创建环境变量

### 基本步骤

1. 进入"环境变量"页面
2. 点击"新建变量"按钮
3. 填写变量信息：
   - **变量名称**：变量的标识名
   - **变量值**：变量的值
   - **描述**：变量的用途说明
4. 点击"保存"完成创建

### 变量命名规范

- 只能包含大写字母、数字和下划线
- 建议使用有意义的名称，如 `API_KEY`、`DB_PASSWORD`
- 不能以数字开头

---

## 管理环境变量

### 编辑变量

1. 在变量列表中，点击变量的"编辑"按钮
2. 修改变量值或描述
3. 点击"保存"

### 删除变量

1. 在变量列表中，点击变量的"删除"按钮
2. 确认删除

> ⚠️ **警告**：删除变量后，使用该变量的云函数可能会出错。

---

## 在云函数中使用

### 读取环境变量

在云函数代码中，环境变量通过 `$env` 参数自动注入，可以直接使用：

```php
// 环境变量通过 $env 参数自动注入
$apiKey = $env['API_KEY'] ?? null;
$dbPassword = $env['DB_PASSWORD'] ?? null;

// 使用环境变量
// ...

return [
    'code' => 200,
    'data' => $result
];
```

### 示例

```php
// 读取 API 密钥
$apiKey = $env['OPENAI_API_KEY'] ?? '';

// 调用第三方 API
$response = $Http->post('https://api.example.com/endpoint', [
    'data' => $payload
]);

return [
    'code' => 200,
    'data' => $response
];
```

## 变量类型

### 普通变量

普通的配置参数，如：

- `APP_NAME`：应用名称
- `TIMEZONE`：时区设置
- `DEBUG_MODE`：调试模式开关

### 敏感变量

敏感信息，如：

- `API_KEY`：第三方 API 密钥
- `DB_PASSWORD`：数据库密码
- `SECRET_KEY`：加密密钥

敏感变量在界面上会以掩码形式显示，保护信息安全。

## 常见问题

### 环境变量会暴露给前端吗？

不会。环境变量只能在后端云函数中使用，不会暴露给前端。

### 环境变量可以共享吗？

不可以。每个项目有独立的环境变量，互不干扰。

### 如何更新环境变量？

在变量列表中，点击"编辑"按钮，修改变量值并保存。更新后，新的变量值会在下次函数执行时生效。
