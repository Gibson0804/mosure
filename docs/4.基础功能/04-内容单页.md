# 内容单页

## 什么是内容单页？

内容单页是一种特殊的内容模型类型，用于只需要一个实例的内容，如"关于我们"、"联系方式"、"服务条款"等。

与内容列表模型相比：

| 特性 | 内容列表 | 内容单页 |
|------|---------|---------|
| 数据量 | 多条记录 | 单条记录 |
| 使用场景 | 文章列表、产品列表等 | 关于我们、联系方式等 |
| 模型编辑 | 拖拽字段定义模型 | 拖拽字段定义模型 |
| 内容管理 | 列表管理，增删改查 | 单页编辑，直接修改 |

### 核心功能

- **拖拽建模**：与内容列表模型一样，通过拖拽字段定义模型结构
- **单页编辑**：直接编辑单条内容，无需列表视图
- **开放 API**：通过 API 获取和更新单页内容
- **灵活应用**：适用于各种静态页面内容

---

## 创建内容单页模型

### 创建步骤

内容单页模型的创建方式与内容列表模型完全相同：

1. 进入"内容模型"页面
2. 点击"新建模型"按钮
3. 在模型类型中选择"内容单页"
4. 拖拽需要的字段到画布
5. 配置字段属性
6. 保存模型

### 模型类型选择

在创建模型时，选择"内容单页"类型：

- **内容列表**：适用于需要多条记录的场景
- **内容单页**：适用于只需要一条记录的场景

> 💡 **提示**：模型类型创建后不可修改，请根据实际需求选择。

---

## 管理内容单页

### 编辑内容

1. 进入"内容模型"页面
2. 找到类型为"内容单页"的模型
3. 点击模型列表中的"内容管理"按钮
4. 直接编辑单页内容
5. 保存更改

### 内容单页的特点

- **唯一实例**：每个内容单页模型只有一条内容记录
- **直接编辑**：无需列表视图，直接进入编辑页面
- **实时保存**：修改后立即生效

## 使用场景

### 典型应用

内容单页适用于以下场景：

| 场景 | 说明 | 示例字段 |
|------|------|---------|
| 公司介绍 | 展示公司基本信息 | 标题、简介、发展历程、团队介绍 |
| 联系方式 | 展示联系信息 | 地址、电话、邮箱、地图 |
| 服务条款 | 展示法律文档 | 条款内容、更新日期 |
| 帮助文档 | 展示使用说明 | 文档内容、分类、更新时间 |
| 活动公告 | 展示活动信息 | 活动标题、时间、地点、详情 |

### 与内容列表的配合

内容单页和内容列表可以配合使用：

- **内容单页**：用于网站的固定页面（关于我们、联系方式）
- **内容列表**：用于动态内容（新闻列表、产品展示）

## 开放 API

### 获取内容单页

**接口地址**：
```
GET /open/page/detail/{table_name}
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "title": "关于我们",
    "content": "公司简介...",
    "created_at": "2024-01-01 00:00:00",
    "updated_at": "2024-01-01 00:00:00"
  }
}
```

### 更新内容单页

**接口地址**：
```
PUT /open/page/update/{table_name}
```

**请求体**：

```json
{
  "title": "更新后的标题",
  "content": "更新后的内容..."
}
```

## 常见问题

### 内容单页和内容列表有什么区别？

- **内容列表**：用于管理多条记录，如文章列表、产品列表
- **内容单页**：用于管理单页内容，如关于我们、联系方式

模型编辑方式完全相同，区别在于使用场景和数据量。

### 可以把内容单页改成内容列表吗？

不可以。模型类型创建后不可修改。如果需要更改，需要创建新模型并迁移数据。

### 内容单页可以有多条记录吗？

技术上可以，但内容单页的设计初衷是只维护一条记录。如果需要多条记录，请使用内容列表模型。

### 如何选择使用内容单页还是内容列表？

- 如果内容需要多条记录（如文章、产品），使用**内容列表**
- 如果内容只需要一条记录（如关于我们、联系方式），使用**内容单页**

### 内容单页支持版本控制吗？

支持。内容单页与内容列表一样，支持内容版本和历史记录功能。
