# 任务化能力与异步体验设计

## 什么是任务化能力？

Mosure 将耗时操作（如批量生成内容、导出数据、AI 处理等）设计为异步任务，用户提交后立即返回，任务在后台执行。这样可以避免长时间等待，提升用户体验。

### 核心价值

- **不阻塞操作**：提交任务后立即返回，无需等待
- **进度可见**：实时查看任务进度和状态
- **可重试取消**：任务失败可重试，不需要的任务可取消
- **日志记录**：完整的任务执行日志，便于排查问题

---

## 任务类型

Mosure 支持多种任务类型：

| 任务类型 | 说明 | 典型场景 |
|---------|------|---------|
| AI 内容生成 | 批量生成内容 | 批量创建文章、产品信息等 |
| AI 建模建议 | AI 辅助生成模型结构 | 创建复杂内容模型 |
| 内容批量处理 | 批量处理内容数据 | 批量更新、批量删除等 |
| AI Agent 运行 | 执行 AI Agent 任务 | 复杂的 AI 自动化任务 |
| 富文本编辑 | AI 辅助编辑富文本 | 优化文章内容、格式调整等 |
| Chrome 采集 AI | AI 辅助网页内容采集 | 智能提取网页内容 |
| 媒体采集 | 媒体资源采集 | 批量下载图片、视频等 |

## 任务状态

任务有完整的生命周期，状态流转如下：

```
pending（待处理）→ running（运行中）→ completed（已完成）
                      ↓
                   failed（失败）
                      ↓
                   cancelled（已取消）
```

### 状态说明

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| pending | 任务已创建，等待执行 | 取消 |
| running | 任务正在执行 | 取消 |
| completed | 任务执行成功 | 查看日志、重新执行 |
| failed | 任务执行失败 | 查看日志、重试 |
| cancelled | 任务已取消 | 重新执行 |

## 任务中心

### 访问任务中心

1. 进入管理后台
2. 点击左侧菜单的"任务中心"
3. 可以看到所有任务的列表

### 任务列表信息

任务列表显示以下信息：

- **任务名称**：任务的描述性名称
- **任务类型**：AI 生成、数据导出等
- **状态**：当前任务状态
- **进度**：任务完成百分比
- **创建时间**：任务创建的时间
- **完成时间**：任务完成的时间（如果已完成）
- **操作**：查看详情、重试、取消等

### 任务详情

点击任务可以查看详细信息：

- **任务信息**：任务类型、参数、状态等
- **执行步骤**：任务分解的步骤列表
- **执行日志**：详细的执行日志，包括错误信息
- **执行结果**：任务执行的结果数据

## 使用场景

### 场景 1：批量生成内容

当你需要批量生成大量内容时：

1. 进入"内容管理"
2. 选择一个模型
3. 点击"批量生成"按钮
4. 设置生成数量和提示词
5. 点击"提交"，系统会创建一个异步任务
6. 在"任务中心"查看任务进度
7. 任务完成后，内容自动保存到数据库

### 场景 2：AI 辅助建模

当你使用 AI 建模时：

1. 进入"内容模型"
2. 点击"AI 建议"按钮
3. 输入需求描述
4. 点击"提交"，系统会创建一个异步任务
5. 在"任务中心"查看任务进度
6. 任务完成后，可以查看生成的模型结构

### 场景 3：Chrome 内容采集

当你需要采集网页内容时：

1. 打开 Chrome 浏览器，安装 Mosure 插件
2. 访问目标网页
3. 点击插件图标，选择"智能采集"
4. 设置采集规则和 AI 处理选项
5. 点击"开始采集"，系统会创建一个异步任务
6. 在"任务中心"查看任务进度
7. 任务完成后，内容自动保存到数据库

## 任务操作

### 查看任务详情

在任务列表中，点击任务名称可以查看详细信息，包括：

- 任务参数
- 执行步骤
- 执行日志
- 执行结果

### 重试失败任务

如果任务执行失败，可以点击"重试"按钮重新执行：

1. 找到失败的任务
2. 点击"重试"按钮
3. 系统会使用相同的参数重新执行任务

### 取消任务

对于正在运行或待处理的任务，可以点击"取消"按钮：

1. 找到要取消的任务
2. 点击"取消"按钮
3. 任务状态变为"已取消"

### 删除任务

对于已完成或已取消的任务，可以删除：

1. 找到要删除的任务
2. 点击"删除"按钮
3. 确认删除

> ⚠️ **注意**：删除任务不会影响任务已经产生的结果（如已生成的内容）。

## 任务日志

### 日志内容

任务日志记录了任务执行的详细信息：

- **开始时间**：任务开始执行的时间
- **结束时间**：任务结束执行的时间
- **执行步骤**：任务分解的步骤及其状态
- **错误信息**：如果任务失败，会记录错误详情
- **输出信息**：任务的输出结果

### 查看日志

1. 进入任务详情页
2. 滚动到"执行日志"部分
3. 可以看到完整的日志信息

### 日志级别

日志包含不同级别的信息：

- **INFO**：一般信息，如步骤开始、步骤完成
- **WARNING**：警告信息，如非致命错误
- **ERROR**：错误信息，如任务失败原因

## 常见问题

### 任务执行失败怎么办？

首先查看任务日志，找出失败原因：

1. 进入任务详情页
2. 查看"执行日志"
3. 找到错误信息
4. 根据错误信息解决问题
5. 点击"重试"按钮重新执行

### 任务一直处于运行状态怎么办？

如果任务长时间处于运行状态，可能是以下原因：

- 任务确实需要较长时间（如批量生成大量内容）
- 任务卡住了（如网络问题、资源不足）

建议：

1. 查看任务日志，确认是否有输出
2. 如果长时间无输出，可以取消任务
3. 检查系统资源（CPU、内存、磁盘空间）
4. 重新提交任务

### 可以同时执行多个任务吗？

可以。系统支持多个任务同时执行，但建议：

- 不要同时提交太多任务，避免资源耗尽
- 优先执行重要任务
- 定期清理已完成或已取消的任务

### 任务会自动重试吗？

不会。任务失败后需要手动重试。这样可以让你先查看失败原因，解决问题后再重试。

### 任务执行会产生费用吗？

任务本身不会产生额外费用，但如果任务调用了付费服务（如 AI 服务），可能会产生相应的费用。

---

## 最佳实践

1. **合理设置参数**：批量生成时，合理设置数量，避免生成过多无用内容
2. **定期清理任务**：定期删除已完成或已取消的任务，保持任务列表整洁
3. **查看任务日志**：任务失败时，及时查看日志，找出问题原因
4. **监控资源使用**：执行大量任务时，注意监控系统资源使用情况
5. **分批处理**：对于大量数据，建议分批处理，避免一次性提交过多任务
