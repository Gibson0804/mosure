# 低代码拖拽表单与字段系统

## 什么是低代码表单系统？

Mosure 的低代码表单系统让你通过简单的拖拽操作就能创建内容模型和表单，完全不需要编写代码。无论你是想创建博客文章、产品信息、活动报名表单，还是其他任何类型的内容，都可以通过可视化界面快速完成。

### 核心功能

- **拖拽式设计**：从左侧面板拖拽字段到中间画布，像搭积木一样简单
- **实时预览**：所见即所得，拖拽的同时就能看到最终效果
- **智能配置**：右侧面板可以快速调整字段属性
- **自动建表**：保存后系统会自动创建对应的数据库表
- **AI 辅助**：用自然语言描述你的需求，AI 帮你生成表单结构

## 支持的字段类型

系统提供了 20 种常用字段类型，满足各种业务场景：

### 基础输入

| 字段 | 用途 | 适合场景 |

| 单行文本 | 短文本输入 | 标题、姓名、地址等 |
| 多行文本 | 长文本输入 | 描述、备注、简介等 |

| 多选框 | 从选项中选择多个 | 兴趣爱好、技能等 |
| 开关 | 是/否切换 | 是否推荐、是否公开等 |

### 数字类

| 字段 | 用途 | 适合场景 |

## 如何使用

### 第一步：创建模型

1. 进入管理后台，点击"内容模型"菜单
2. 点击"新建模型"按钮
3. 你会看到一个三栏布局的编辑器：
   - **上侧**：字段组件面板，包含所有可用的字段类型
   - **中间**：表单画布，实时预览表单效果
   - **右侧**：配置面板，设置模型和字段属性

### 第二步：拖拽添加字段

从上侧面板拖拽你需要的字段到中间画布：

- 拖拽"单行文本"到画布，添加标题字段
- 拖拽"富文本"到画布，添加内容字段
- 拖拽"图片上传"到画布，添加封面图字段
- 继续添加你需要的其他字段

每个字段都会自动生成一个唯一的标识，你可以在右侧面板查看和修改。

### 第三步：配置字段属性

点击画布中的任意字段，右侧面板会显示该字段的配置选项：

- **标签**：字段的显示名称，比如"文章标题"
- **字段名**：系统内部使用的标识，会自动规范化
- **长度**：字段在表单中的宽度，可以调整
- **是否必填**：勾选后该字段为必填项
- **选项配置**：对于选择类字段，可以设置选项列表

### 第四步：设置模型信息

在右侧面板顶部，设置模型的基本信息：

- **页面名称**：模型的名称，比如"文章模型"
- **标识ID**：模型的唯一标识，保存后不可修改
- **模型类型**：选择"内容列表"或"内容单页"

### 第五步：保存模型

点击右上角的"保存"按钮，系统会：

1. 验证表单配置是否完整
2. 自动创建对应的数据库表
3. 保存模型定义

保存后，你就可以在"内容管理"中使用这个模型来创建内容了。

## 高级功能

### 动态选项

对于下拉选择、单选框、多选框等字段，你可以选择两种选项来源：

**静态选项**：手动输入选项列表，适合固定的选项，比如"男/女"、"已发布/草稿"等。

**动态模型选项**：从其他模型中动态拉取选项，比如从"分类模型"中拉取分类列表。这样当分类发生变化时，选项会自动更新。

### AI 辅助生成

如果你不确定需要哪些字段，可以让 AI 帮你设计：

1. 在模型列表页点击"AI 建议"
2. 用自然语言描述你的需求，比如：
   - "创建一个博客文章模型，包含标题、内容、分类、发布时间、封面图"
   - "产品信息模型，需要商品名称、价格、库存、规格、详情"
3. 系统会自动生成表单结构，你可以在此基础上调整

### 字段拖拽排序

在中间画布中，你可以拖拽字段来调整它们的顺序。保存后，这个顺序会应用到内容编辑页面。

## 常见问题

### 标识ID是什么？

标识ID是模型的唯一标识，对应数据库中的表名。创建后不可修改，因为修改会导致已有数据无法访问。建议使用有意义的英文或拼音，比如 `article`、`product`、`news`。

### 字段名可以修改吗？

可以修改字段的显示标签，但字段名（系统内部标识）建议不要频繁修改，特别是在已有数据的情况下。如果必须修改，系统会自动处理数据迁移。

### 删除模型会丢失数据吗？

删除模型前，系统会检查是否有关联的内容数据。如果有数据，会提示你先处理数据再删除模型，避免误操作导致数据丢失。

### 如何设置必填字段？

在右侧配置面板中，勾选"是否必填"即可。必填字段在内容编辑时会显示红色星号，提交时会验证是否填写。

### 字段长度是什么意思？

字段长度指的是字段在表单中的显示宽度，范围是 4-24，不影响数据库中实际存储的字符长度。你可以根据字段的重要性调整宽度，让重要字段更突出。

## 使用场景示例

### 博客文章模型

创建一个博客文章模型，包含以下字段：

- 标题（单行文本，必填）
- 摘要（多行文本）
- 内容（富文本，必填）
- 分类（下拉选择，动态从分类模型拉取）
- 标签（标签输入）
- 封面图（图片上传）
- 发布时间（日期时间）
- 是否推荐（开关）

### 产品信息模型

创建一个产品信息模型，包含以下字段：

- 产品名称（单行文本，必填）
- 产品编号（单行文本）
- 价格（数字输入，必填）
- 库存（数字输入）
- 规格（下拉选择）
- 详情（富文本）
- 产品图（图片上传，支持多图）
- 上架时间（日期时间）
- 是否上架（开关）

### 活动报名表单

创建一个活动报名表单，包含以下字段：

- 姓名（单行文本，必填）
- 手机号（单行文本，必填）
- 邮箱（单行文本）
- 性别（单选框）
- 年龄（数字输入）
- 报名时间（日期时间，自动填充）
- 备注（多行文本）

## 小贴士

- **从简单开始**：先创建一个包含基本字段的模型，保存后再逐步添加复杂字段
- **善用AI**：不确定需要哪些字段时，先用AI生成一个基础结构，再根据需求调整
- **合理分组**：使用分割线将相关字段分组，让表单更清晰
- **必填字段**：只对真正必要的字段设置必填，避免影响用户体验
- **预览效果**：保存前多在画布中预览，确保字段顺序和布局符合预期
