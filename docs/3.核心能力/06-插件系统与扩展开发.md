# 插件系统与扩展开发

## 什么是插件系统？

Mosure 的插件系统让你能够通过安装、启用插件来扩展系统功能，而无需修改核心代码。插件可以提供自定义内容模型、云函数、菜单、配置变量、定时任务和触发器等功能，让 Mosure 能够适应各种业务场景。

### 核心功能

- **模块化扩展**：插件独立于核心系统，安装卸载不影响系统稳定性
- **安全隔离**：插件使用独立的数据表和目录结构，避免冲突
- **热插拔**：支持动态启用/禁用插件，无需重启系统
- **标准化接口**：统一的插件接口，开发简单规范

## 插件能力

一个 Mosure 插件可以提供以下能力：

### 内容模型

插件可以定义自定义的内容模型，用于存储特定业务的数据：

- 定义模型字段和类型
- 设置模型属性（是否必填、默认值等）
- 提供模型配置和验证规则

### 云函数

插件可以提供 Web API 和触发函数：

- **Web 函数**：对外暴露的 HTTP 接口
- **触发函数**：响应系统事件的函数（如内容发布时触发）

### 菜单

插件可以添加自定义菜单项到管理界面：

- 菜单名称和图标
- 菜单链接和路由
- 菜单权限控制

### 配置变量

插件可以定义可配置的变量，让用户自定义插件行为：

- 变量名称和类型
- 默认值和验证规则
- 变量分组和说明

### 定时任务

插件可以定义计划任务，自动执行定期操作：

- Cron 表达式
- 任务执行逻辑
- 任务日志记录

### 触发器

插件可以定义事件响应机制，在特定事件发生时执行操作：

- 事件类型（内容新增/更新/删除、函数执行等）
- 触发条件
- 执行逻辑

## 插件管理

### 查看插件列表

进入"系统管理" → "插件管理"，可以看到所有可用插件：

- 插件名称和描述
- 插件版本
- 安装状态（未安装/已安装/已启用）
- 提供的资源数量（模型数、函数数、菜单数等）

### 安装插件

1. 在插件列表中找到要安装的插件
2. 点击"安装"按钮
3. 系统会执行插件的安装逻辑：
   - 创建必要的数据表
   - 注册路由和监听器
   - 初始化配置
4. 安装成功后，插件状态变为"已安装"

> ⚠️ **注意**：安装过程在事务中执行，如果失败会自动回滚，不会影响系统稳定性。

### 启用插件

插件安装后默认是禁用状态，需要手动启用：

1. 在插件列表中找到已安装的插件
2. 点击"启用"按钮
3. 系统会注册插件提供的所有资源（模型、函数、菜单等）
4. 启用成功后，插件功能即可使用

### 禁用插件

如果暂时不需要某个插件，可以禁用它：

1. 在插件列表中找到已启用的插件
2. 点击"禁用"按钮
3. 系统会注销插件提供的所有资源
4. 插件状态变为"已安装（已禁用）"

### 卸载插件

如果不再需要某个插件，可以卸载它：

1. 在插件列表中找到已安装的插件
2. 点击"卸载"按钮
3. 系统会询问是否删除插件数据：
   - **保留数据**：只卸载插件，保留数据表和数据
   - **删除数据**：卸载插件并删除所有相关数据
4. 确认后执行卸载操作

> ⚠️ **注意**：卸载操作不可逆，请谨慎选择是否删除数据。

## 插件开发

### 插件目录结构

一个标准的 Mosure 插件目录结构如下：

```
app/Plugins/YourPlugin/
├── YourPlugin.php              # 插件主类
├── config/
│   ├── variables.json          # 配置变量定义
│   ├── triggers.json           # 触发器定义
│   ├── schedules.json          # 定时任务定义
│   └── menu.json               # 菜单定义
├── models/                     # 自定义模型
│   ├── Model1.php
│   └── Model2.php
├── functions/                  # 云函数
│   ├── web/                    # Web 函数
│   │   └── Function1.php
│   └── triggers/               # 触发函数
│       └── Trigger1.php
└── README.md                   # 插件说明文档
```

### 插件主类

插件主类必须实现 `PluginInterface` 接口：

```php
<?php

namespace App\Plugins\YourPlugin;

use App\Plugins\AbstractPlugin;

class YourPlugin extends AbstractPlugin
{
    /**
     * 插件唯一标识
     */
    public function getId(): string
    {
        return 'your_plugin';
    }

    /**
     * 插件名称
     */
    public function getName(): string
    {
        return '你的插件';
    }

    /**
     * 插件描述
     */
    public function getDescription(): string
    {
        return '这是一个示例插件';
    }

    /**
     * 插件版本
     */
    public function getVersion(): string
    {
        return '1.0.0';
    }

    /**
     * 安装前执行
     */
    public function onBeforeInstall(): void
    {
        // 安装前的准备工作
    }

    /**
     * 安装后执行
     */
    public function onAfterInstall(): void
    {
        // 安装后的初始化工作
    }

    /**
     * 卸载前执行
     */
    public function onBeforeUninstall(): void
    {
        // 卸载前的清理工作
    }

    /**
     * 卸载后执行
     */
    public function onAfterUninstall(): void
    {
        // 卸载后的清理工作
    }
}
```

    {
      "key": "enable_cache",

### 触发器定义

在 `config/triggers.json` 中定义触发器：

```json
{
  "triggers": [
    {
      "name": "on_content_publish",
      "label": "内容发布时触发",
      "description": "当内容发布时自动执行",
      "event_type": "content.update",
      "conditions": {
        "hook_target": "content",
        "hook_mold_id": "*"
      }
    }
  ]
}
```

### 定时任务定义

在 `config/schedules.json` 中定义定时任务：

```json
{
  "schedules": [
    {
      "name": "daily_sync",
      "label": "每日同步",
      "description": "每天凌晨 2 点执行数据同步",
      "cron": "0 2 * * *",
      "function": "daily_sync_task"
    }
  ]
}
```

### 菜单定义

在 `config/menu.json` 中定义菜单：

## 插件生命周期

### 安装流程

1. 调用 `onBeforeInstall()` 钩子
2. 创建插件数据表
3. 注册插件配置
4. 调用 `onAfterInstall()` 钩子
5. 记录安装信息到数据库

### 启用流程

1. 调用 `enable()` 方法
2. 注册内容模型
3. 注册云函数
4. 注册菜单
5. 注册定时任务
6. 注册触发器
7. 更新插件状态为"已启用"

### 禁用流程

1. 调用 `disable()` 方法
2. 注销内容模型
3. 注销云函数
4. 注销菜单
5. 注销定时任务
6. 注销触发器
7. 更新插件状态为"已禁用"

### 卸载流程

1. 调用 `onBeforeUninstall()` 钩子
2. 删除插件数据（根据用户选择）
3. 删除插件配置
4. 调用 `onAfterUninstall()` 钩子
5. 删除安装记录

## 安全机制

### 事务保护

所有插件操作都在数据库事务中执行：

- 安装失败自动回滚，不会留下残留数据
- 卸载失败自动回滚，不会破坏系统数据
- 确保系统始终处于一致状态

### 权限控制

插件可以定义自己的权限规则：

- 菜单项可以设置访问权限
- 云函数可以设置调用权限
- 配置变量可以设置编辑权限

### 错误处理

系统提供完善的错误处理机制：

- 捕获所有异常并记录日志
- 向用户显示友好的错误提示
- 失败操作自动回滚

### 最小侵入

插件采用最小侵入设计：

- 使用独立的数据表，不修改核心表
- 使用独立的目录结构，不修改核心文件
- 通过接口和事件与核心系统交互

## 常见问题

### 插件会影响系统性能吗？

插件只在启用时才会加载资源，禁用后不会占用系统资源。建议只启用必要的插件。

### 插件可以修改核心代码吗？

不可以。插件只能通过提供的接口扩展功能，不能直接修改核心代码，这保证了系统的稳定性。

### 插件数据会丢失吗？

卸载插件时可以选择是否保留数据。如果选择保留数据，数据表和数据都会保留，下次安装可以继续使用。

### 如何调试插件？

1. 检查插件日志：`storage/logs/plugin.log`
2. 使用 Laravel 的调试工具查看执行过程
3. 在插件代码中添加日志记录

### 插件可以访问其他插件的数据吗？

可以。插件可以通过系统提供的 API 访问其他插件的数据，但建议保持插件间的独立性。

### 插件支持版本升级吗？

支持。插件可以定义升级逻辑，系统会在检测到新版本时自动执行升级。

## 最佳实践

### 插件命名

- 使用有意义的名称，如 `BlogPlugin`、`EcommercePlugin`
- 避免使用通用名称，如 `MyPlugin`、`TestPlugin`
- 命名遵循大驼峰规范

### 数据表设计

- 使用插件前缀作为表名前缀，避免冲突
- 合理设计索引，提高查询性能
- 考虑数据迁移和升级

### 错误处理

- 使用 try-catch 捕获异常
- 记录详细的错误日志
- 向用户返回友好的错误提示

### 文档编写

- 提供清晰的 README 文档
- 说明插件的安装、配置和使用方法
- 提供示例代码和最佳实践

### 版本管理

- 使用语义化版本号（如 1.0.0、1.1.0、2.0.0）
- 记录版本变更日志
- 提供升级指南

## 总结

Mosure 的插件系统提供了一种安全、灵活的方式来扩展系统功能。通过插件，你可以：

- 快速添加业务功能
- 保持核心系统的稳定性
- 实现功能的模块化管理
- 降低维护成本

无论是开发自己的插件，还是使用社区插件，都能让 Mosure 更好地满足你的业务需求。
