# 插件前端页面托管设计

## 需求背景

作为无头 CMS，用户需要快速部署前端页面来展示内容。为了降低部署门槛，在现有插件系统基础上扩展，允许插件包含前端页面，安装后可直接访问。

## 设计目标

1. **复用现有系统**：基于现有插件架构，无需独立模块
2. **简单易用**：插件安装即用，无需额外配置
3. **灵活可控**：支持多插件、访问控制、自定义路由
4. **安全可靠**：文件隔离、访问限制、防篡改

## 架构设计

### 方案：插件扩展

在现有插件系统基础上扩展，支持插件包含前端页面：

```
Plugins/
└── {plugin_name}/
    └── {version}/
        ├── plugin.json          # 插件配置
        ├── {plugin_name}.php    # 插件主类
        ├── README.md            # 插件说明
        ├── data/                # 数据导入（可选）
        ├── frontend/            # 前端页面（可选）
        ├── functions/           # 云函数（可选）
        │   ├── endpoints/       # Web 函数
        │   ├── hooks/           # 触发函数
        │   └── variables.json   # 环境变量
        ├── menus/               # 菜单（可选）
        ├── models/              # 内容模型（可选）
        └── snapshot/            # 快照（可选）
```

### plugin.json 配置

```json
{
  "id": "blog_theme_v1",
  "name": "博客主题",
  "description": "博客主题插件",
  "author": "Mosure Team",
  "version": "v1",
  "has_frontend": true,
  "has_src": false,
  "provides": {
    "models": [],
    "functions": {
      "endpoints": [],
      "hooks": [],
      "variables": false,
      "triggers": false,
      "schedules": false
    },
    "data": [],
    "menus": []
  }
}
```

### 前端清单 manifest.json

```json
{
  "package_version": "v1",
  "plugin": "todolist",
  "build_time": "2025-01-15T12:00:00+00:00",
  "files": [
    "index.html",
    "config.js",
    "assets/index-s1upIyYY.js",
    "assets/index-BmBWFn4J.css"
  ],
  "size": 1024000
}
```

### 前端目录结构

```
frontend/
└── dist/
    ├── index.html          # 入口 HTML 文件
    ├── config.js           # API 配置文件
    ├── assets/             # 静态资源
    │   ├── index-*.js      # 打包后的 JS 文件
    │   └── index-*.css     # 打包后的 CSS 文件
    └── manifest.json       # 文件清单
```

### index.html
index.hmtl为前端页面的入口文件，如果是非纯静态页面（需要调用后端服务），必须包含 `<script src="config.js"></script>` 以加载 API 配置

### config.js 格式

```javascript
// API 配置文件（打包后可直接修改）
window.API_CONFIG = {
  domain: "{$domain}",
  apiKey: "{$apiKey}"
}
```

**说明**：
- `{$domain}` 和 `{$apiKey}` 是占位符，会在运行时被替换为实际值
- 前端代码通过 `window.API_CONFIG.domain` 和 `window.API_CONFIG.apiKey` 获取配置
- 支持在打包后手动修改配置文件

## 核心功能

### 1. 插件安装流程

```
上传/安装插件
  ↓
解析 plugin.json
  ↓
检测 has_frontend
  ↓
复制 frontend/ 到 storage/frontend/{plugin_name}/
  ↓
注册前端路由
  ↓
完成安装
```

### 2. 前端页面访问

**访问路径**：
```
/frontend/{project_prefix}/{plugin}/index.html
```

**示例**：
- 项目前缀：`blog`
- 插件名称：`theme`
- 访问地址：`https://domain.com/frontend/blog/theme/index.html`


## 性能优化

当前实现的性能优化：
1. **缓存策略**：设置 1 天的浏览器缓存头（`Cache-Control: public, max-age=86400`）

## 总结

### 优点
1. **复用现有架构**：无需独立模块，降低开发成本
2. **统一管理**：插件和前端页面统一管理，用户体验好
3. **灵活扩展**：插件可以同时提供后端和前端功能
4. **易于维护**：基于现有插件系统，维护成本低

### 缺点
基于现有插件系统扩展前端托管功能是简单方案，适合小型应用或快速验证想法落地，如果需要提供更好的服务，建议使用独立的前端托管方案。
