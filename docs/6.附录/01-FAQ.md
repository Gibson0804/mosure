# FAQ

## 安装与配置

### 如何安装 Mosure？

1. 下载 Mosure 源码
2. 执行 `./bin/install.sh`，Windows 使用 `bin\install.bat`
3. 或手动执行 `composer install`
4. 再执行 `php artisan mosure:install`
5. 访问 `/login` 登录后台

### 安装向导无法访问？

检查以下几点：

- 确认已运行 `php artisan serve`
- 确认 `.env` 文件已配置
- 检查 SQLite 文件或 MySQL 连接是否正常
- 查看日志文件 `storage/logs/laravel.log`

### 如何升级 Mosure？

1. 备份数据库和文件
2. 拉取最新代码：`git pull`
3. 更新依赖：`composer update` 和 `npm install`
4. 运行迁移：`php artisan migrate`
5. 构建前端：`npm run build`
6. 重启服务

## API 相关

### 如何获取 API 密钥？

1. 进入"API 与密钥"页面
2. 点击"新建密钥"按钮
3. 填写密钥名称和描述
4. 点击"创建"
5. 复制生成的密钥（只显示一次）

### API 调用返回 401 错误？

检查以下几点：

- API 密钥是否正确
- API 密钥是否已启用
- Header 是否正确传递：`X-API-Key: your_key` 或 `Authorization: Bearer your_key`
- 项目前缀是否正确

### 如何调用 Open API？

使用 API 密钥调用：

```bash
curl -X GET "http://your-domain/open/blog/content/list/article" \
  -H "X-API-Key: your_api_key"
```

或使用 Bearer Token：

```bash
curl -X GET "http://your-domain/open/blog/content/list/article" \
  -H "Authorization: Bearer your_api_key"
```

## MCP 相关

### 如何获取 MCP Token？

1. 进入"项目配置"页面
2. 找到"MCP 配置"部分
3. 点击"生成 Token"按钮
4. 复制生成的 Token（只显示一次）

### MCP 调用返回 401 错误？

检查以下几点：

- MCP Token 是否正确
- MCP 服务是否已启用
- Header 是否正确传递：`Authorization: Bearer your_token`
- 项目前缀 Header 是否正确：`X-Project-Prefix: blog`

### 如何调用 MCP 工具？

使用mcp客户端配置

## 内容管理

### 如何创建内容模型？

1. 进入"内容类型与字段"页面
2. 点击"新建模型"按钮
3. 使用拖拽方式添加字段
4. 配置字段属性
5. 保存模型

### 内容模型创建后无法使用？

检查以下几点：

- 模型是否已保存
- 模型是否已启用
- 数据表是否已创建（检查数据库）
- 清除缓存：`php artisan cache:clear`

### 如何回滚内容版本？

1. 进入"内容版本与历史"页面
2. 找到要回滚的版本
3. 点击"回滚"按钮
4. 确认回滚

## 云函数

### 如何创建 Web 函数？

1. 进入"Web 函数"页面
2. 点击"新建函数"按钮
3. 填写函数名称和描述
4. 编写函数代码
5. 保存函数

### 云函数调用失败？

检查以下几点：

- 函数代码是否有语法错误
- 函数是否已启用
- 查看执行日志
- 检查环境变量是否正确

### 如何调试云函数？

使用在线编辑器的调试功能：

1. 打开函数编辑页面
2. 点击"测试"按钮
3. 输入测试参数
4. 查看执行结果和日志

## 媒体资源

### 文件上传失败？

检查以下几点：

- 文件大小是否超过限制（默认 100MB）
- 文件类型是否允许
- 存储空间是否足够
- 查看错误日志

### 如何批量上传文件？

1. 进入"媒体资源"页面
2. 点击"上传"按钮
3. 选择多个文件
4. 点击"开始上传"

## 项目管理

### 如何创建新项目？

1. 进入"项目与工作区"页面
2. 点击"新建项目"按钮
3. 填写项目名称和前缀
4. 点击"创建"

### 项目前缀可以修改吗？

创建后项目前缀不可修改。如需修改，建议创建新项目并迁移数据。

### 如何删除项目？

1. 进入"项目与工作区"页面
2. 找到要删除的项目
3. 点击"删除"按钮
4. 确认删除

> ⚠️ **警告**：删除项目会删除所有相关数据，请谨慎操作。

## 系统配置

### 如何修改系统配置？

1. 点击左上角的系统菜单
2. 点击"系统配置"
3. 修改配置项
4. 点击"保存"

### 修改配置后不生效？

尝试以下操作：

1. 清除缓存：`php artisan cache:clear`
2. 重启服务
3. 检查配置是否正确

## 常见错误

### 500 服务器错误

查看日志文件 `storage/logs/laravel.log`，找出错误原因。

### 404 页面不存在

检查路由是否正确，URL 是否拼写正确。

### 422 验证失败

检查请求参数是否符合要求，查看错误提示。
