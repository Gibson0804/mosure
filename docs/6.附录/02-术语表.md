# 术语表

## 项目相关

### Project（项目）

Mosure 的基本工作单位，每个项目有独立的数据和配置。

**示例**：博客项目、电商项目、企业官网项目

### Prefix（项目前缀）

项目的唯一标识，用于区分不同项目，只能包含小写字母、数字和下划线。

**示例**：`blog`、`shop`、`company`

---

## 内容相关

### Mold（内容模型/内容类型）

定义内容的结构和字段类型，类似于数据库表结构。

**示例**：文章模型、产品模型、用户模型

### Content（内容）

基于内容模型创建的具体数据实例。

**示例**：一篇具体的文章、一个具体的产品

### Subject/Page/content_single（主题/单页主题/单页内容）

单页内容的主题配置，用于自定义单页的展示方式。

**示例**：首页主题、关于我们页面主题

---

## 媒体相关

### Media（媒体资源）

上传到系统的文件，包括图片、视频、音频、文档等。

**示例**：文章配图、产品视频、PDF 文档

### MediaFolder（媒体文件夹）

用于组织和管理媒体资源的文件夹。

**示例**：2024年文件夹、产品图片文件夹

### MediaTag（媒体标签）

用于分类和标记媒体资源的标签。

**示例**：风景、人物、产品图

---

## 云函数相关

### Cloud Function（云函数）

在云端运行的代码，分为 Web 函数和触发函数。

**示例**：订单处理函数、邮件发送函数

### Web Function（Web 函数）

通过 HTTP 请求调用的云函数，类似于 API 端点。

**示例**：`/open/blog/func/process_order`

### Trigger Function（触发函数）

在特定事件发生时自动触发的云函数。

**示例**：内容创建时触发、内容更新时触发

---

## 自动化相关

### Trigger（触发器）

定义特定条件下自动执行操作的规则。

**示例**：内容发布时发送通知、内容删除时清理缓存

### Cron（定时任务）

定时任务的调度配置。

**示例**：`0 0 * * *` 表示每天凌晨执行

---

## 插件相关

### Plugin（插件）

扩展系统功能的模块，可以添加自定义功能。

**示例**：博客插件、电商插件、SEO 插件

### Plugin Manager（插件管理器）

负责插件的发现、安装、卸载、启用、禁用的系统组件。

---

## API 相关

### Open API（开放 API）

提供给外部系统调用的 API，需要 API 密钥鉴权。

**示例**：`/open/blog/content/list/article`

### Client API（客户端 API）

提供给前端应用调用的 API，需要客户端 Token 鉴权。

**示例**：`/client/me`

### API Key（API 密钥）

用于访问 Open API 的密钥，每个项目可以有多个 API 密钥。

### Client Token（客户端 Token）

用于访问 Client API 的令牌，每个用户有一个客户端 Token。

---

## MCP 相关

### MCP（Model Context Protocol）

模型上下文协议，允许 AI 助手访问系统的数据和功能。

### MCP Server（MCP 服务器）

Mosure 提供的 MCP 服务，提供 Tools 和 Resources。

### MCP Token（MCP 令牌）

用于访问 MCP 服务的密钥。

### Tool（工具）

AI 助手可以调用的 MCP 工具。

**示例**：`list_contents`、`get_content`

### Resource（资源）

AI 助手可以访问的 MCP 资源。

**示例**：`content://article`、`model://product`

---

## 系统相关

### Audit Log（审计日志）

记录系统重要操作的日志，用于追踪和审计。

**示例**：用户登录、内容创建、配置修改

### SysTask（系统任务）

异步执行的系统任务。

**示例**：数据导出、批量处理、定时同步

### SysTaskStep（系统任务步骤）

系统任务的执行步骤，记录任务的执行进度。
