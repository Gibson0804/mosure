# 目录结构

本文档用于说明仓库关键目录与文件的用途（以当前代码库实际结构为准）。

## 项目根目录

- `app/`：后端业务代码（Controller/Service/Model/Middleware/MCP 等）
- `bootstrap/`：应用启动与引导相关
- `config/`：Laravel 配置文件（app/auth/database/filesystems/queue/session 等）
- `database/`：数据库迁移、种子、工厂
- `docs/`：项目文档
- `public/`：Web 入口与静态资源目录（生产部署 Web Root 指向此目录）
- `bin/`：脚本文件
- `Plugins/`：插件目录
- `resources/`：前端资源与视图（JS/TSX、CSS、Blade 等）
- `routes/`：路由定义（web/open/client/auth/ai 等）
- `storage/`：运行时存储（日志、缓存、上传文件、导出文件等）
- `tests/`：测试
- `vendor/`：Composer 安装的 PHP 依赖
- `node_modules/`：npm 安装的前端依赖

## 常见配置文件

- `.env.example`：环境变量示例文件
- `.env`：实际运行环境变量（安装向导会生成/更新）
- `.env.docker.example`：Docker 环境变量示例
- `composer.json`：PHP 依赖与脚本（如 `composer test`、`composer pint` 等）
- `package.json`：前端依赖与构建脚本
- `docker-compose.yml`、`Dockerfile`：容器化部署相关

## routes/ 路由文件

- `routes/web.php`：管理端路由
- `routes/auth.php`：登录/登出/密码找回/个人资料等
- `routes/open.php`：Open API（`/open/{prefix}`）
- `routes/client.php`：移动端APP使用接口
- `routes/chrome.php`：Chrome插件专用接口
