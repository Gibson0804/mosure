# 快速开始

## 目标

在本地跑起来，并完成一次"建模 → 录入内容 → 调用接口"的最小闭环，体验 Mosure 的核心功能。

---

## 前置条件

在开始之前，请确保你已经：

1. 完成了 Mosure 的安装（参考《安装》文档）
2. 启动了开发环境（推荐执行 `./bin/start.sh` 或 `bin/start.bat`）
3. 完成了安装向导（创建了管理员账号）

---

## 第一步：创建项目

1. 访问管理后台：`http://127.0.0.1:9445/login`
2. 使用安装时创建的管理员账号登录
3. 进入"项目管理"页面，点击"新建项目"
4. 填写项目信息：
   - **项目名称**：例如"我的博客"
   - **项目前缀**：系统会自动根据项目名称生成拼音首字母，也可以手动修改（只能包含小写字母、数字和下划线）
5. 点击"保存"完成创建

> 💡 **提示**：项目前缀是项目的唯一标识，创建后不可修改。建议使用有意义的英文或拼音，比如 `blog`、`news`、`shop`。

---

## 第二步：选择项目

创建项目后，需要选择进入该项目才能开始操作：

1. 在项目列表中，点击刚创建的项目卡片上的"进入"按钮
2. 进入项目后，你会看到仪表盘，显示项目的基本统计信息

> 💡 **提示**：Mosure 采用"项目隔离"的设计，每个项目的内容模型、数据、配置都是独立的。

---

## 第三步：创建内容模型

现在我们来创建一个"文章"模型：

1. 点击左侧菜单的"内容模型"
2. 点击"新建模型"按钮
3. 进入表单构建器，拖拽添加字段：
   - 拖拽"单行文本"到画布，标签改为"标题"，勾选"是否必填"
   - 拖拽"富文本"到画布，标签改为"内容"，勾选"是否必填"
   - 拖拽"下拉选择"到画布，标签改为"分类"
   - 拖拽"图片上传"到画布，标签改为"封面图"
4. 在右侧面板设置模型信息：
   - **页面名称**：文章模型
   - **标识ID**：article（系统会自动生成，建议使用英文）
   - **模型类型**：内容列表
5. 点击"保存"完成创建

> 💡 **提示**：保存后，系统会自动创建对应的数据库表，你可以在"内容管理"中使用这个模型来创建内容了。

---

## 第四步：录入内容

模型创建完成后，我们来添加几篇文章：

1. 点击左侧菜单的"内容管理"
2. 选择刚创建的"文章模型"
3. 点击"新建内容"按钮
4. 填写表单：
   - **标题**：我的第一篇文章
   - **内容**：这是文章的正文内容，支持富文本编辑
   - **分类**：选择一个分类（如果还没有分类，可以手动输入）
   - **封面图**：上传一张图片
5. 点击"保存"完成创建
6. 重复以上步骤，再添加几篇文章

> 💡 **提示**：内容支持草稿和发布状态，创建后默认为草稿，可以点击"发布"按钮发布内容。

---

## 第五步：调用开放 API

现在我们通过 API 来获取文章列表。

### 1. 创建 API 密钥

1. 点击左侧菜单的"API 与密钥"
2. 点击"新建密钥"按钮
3. 填写密钥信息：
   - **密钥名称**：测试密钥
   - **权限范围**：根据需要选择
4. 点击"保存"，系统会生成一个密钥字符串，请妥善保存

### 2. 调用内容列表接口

使用你喜欢的 HTTP 客户端（如 Postman、curl 或浏览器开发者工具）调用接口：

**接口地址**：
```
GET /open/{project_prefix}/content/list/{table_name}
```

**示例**（假设项目前缀为 `blog`，表名为 `article`）：
```bash
curl -X GET "http://127.0.0.1:9445/open/blog/content/list/article" \
  -H "X-API-Key: your_api_key_here"
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": 1,
        "title": "我的第一篇文章",
        "content": "这是文章的正文内容...",
        "category": "技术",
        "cover_image": "http://127.0.0.1:9445/storage/...",
        "created_at": "2024-01-01 12:00:00"
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 20
  }
}
```

### 3. 调用函数网关接口（可选）

如果你创建了云函数，也可以通过函数网关调用：

**接口地址**：
```
POST /open/{project_prefix}/func/{slug}
```

**示例**：
```bash
curl -X POST "http://127.0.0.1:9445/open/blog/func/hello" \
  -H "X-API-Key: your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"name": "Mosure"}'
```

> 💡 **提示**：关于 API 的详细说明（鉴权方式、请求参数、响应格式等），请参考《API 与密钥》和《API 文档》章节。
