# 贡献指南

感谢你对 Mosure 的关注！我们欢迎任何形式的贡献，包括代码、文档、问题报告和功能建议。

---

## 如何贡献

### 报告问题

如果你发现了 bug 或有功能建议：

1. 检查 GitHub/Gitee Issues，确认问题是否已被报告
2. 如果是新问题，创建新的 Issue
3. 提供详细的问题描述，包括：
   - Mosure 版本
   - PHP 版本
   - 复现步骤
   - 预期行为
   - 实际行为
   - 错误信息（如果有）

### 提交代码

#### 分支策略

- `main`：主分支，只接受合并
- `develop`：开发分支
- `feature/*`：功能分支
- `bugfix/*`：修复分支

#### 提交规范

提交信息应该清晰描述你的变更：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型（type）**：
- `feat`：新功能
- `fix`：修复 bug
- `docs`：文档更新
- `style`：代码格式（不影响功能）
- `refactor`：重构
- `test`：添加测试
- `chore`：构建过程或辅助工具的变动

**示例**：

```
feat(content): add content version control

- Add content version tracking
- Support version comparison
- Add version rollback functionality

Closes #123
```

#### Pull Request 流程

1. 从 `develop` 分支创建新分支：
   ```bash
   git checkout develop
   git pull upstream develop
   git checkout -b feature/your-feature-name
   ```
2. 进行开发并提交代码
3. 推送到你的 Fork：
   ```bash
   git push origin feature/your-feature-name
   ```
4. 在 GitHub 上创建 Pull Request
5. 等待代码审查
6. 根据反馈修改代码
7. 合并到 `develop` 分支

### 改进文档

文档是项目的重要组成部分，欢迎改进文档：

1. Fork 项目仓库
2. 修改文档
3. 提交 Pull Request

## 代码规范

### PHP 代码规范

遵循 PSR-12 编码标准：

- 使用 4 个空格缩进
- 类名使用 PascalCase
- 方法名使用 camelCase
- 常量使用 UPPER_CASE

### JavaScript/TypeScript 代码规范

- 使用 2 个空格缩进
- 使用单引号
- 使用 const/let，避免 var
- 使用箭头函数

### 注释规范

- 公共方法必须添加注释
- 复杂逻辑需要添加解释性注释
- 避免无用的注释

## 行为准则

### 我们的承诺

为了营造开放和友好的环境，我们承诺：

- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 不可接受的行为

- 使用性化的语言或图像
- 人身攻击或侮辱性评论
- 公开或私下骚扰
- 未经许可发布他人的私人信息
- 其他不专业或不恰当的行为
