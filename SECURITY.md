# Security Policy

感谢你帮助 Mosure 提升安全性。

## 报告安全问题

请不要在公开 Issue 中直接披露安全漏洞细节。

建议通过以下方式私下联系维护者：

- Gitee Issues：仅在不包含利用细节时提交
- 或通过仓库维护者提供的私下联系方式沟通

如果你提交的是安全漏洞，请尽量提供：

- 影响版本
- 漏洞类型与影响范围
- 复现步骤
- 可能的利用条件
- 修复建议（如有）

## 处理流程

维护者收到报告后会尽快：

1. 确认问题
2. 评估影响范围
3. 制定修复方案
4. 在修复完成后公开说明

## 安全建议

- 生产环境务必关闭 `APP_DEBUG`
- 不要提交真实密钥、密码、令牌到仓库
- 定期执行依赖安全检查，例如：

```bash
composer audit --locked
npm audit
```
